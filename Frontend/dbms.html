Database Management Systems (DBMS)

Module 1: Introduction to DBMS

Learning Objectives: Understand fundamental database concepts, compare DBMS with file systems, and explore database architecture.

1.1 What is a Database?

A database is an organized collection of interrelated data stored with minimal redundancy and accessed by multiple users simultaneously.
Key Characteristics:
  • Persistent storage
  • Self-describing nature (metadata)
  • Program-data independence
  • Multiple user interfaces
File System vs DBMS:
Criteria File System DBMS
Data Redundancy High Minimal
Data Consistency Poor Enforced
Concurrent Access Difficult Managed
Security Limited Granular

1.2 DBMS Architecture

Three-Schema Architecture:
  1. Internal Level: Physical storage (files, indexes)
  2. Conceptual Level: Logical structure (tables, relationships)
  3. External Level: User views (customized presentations)
Data Independence: The ability to modify one level without affecting others:
  • Physical: Change storage without changing schema
  • Logical: Change schema without changing applications

Module 2: Data Models & Relational Databases

Learning Objectives: Understand different data models, master relational model concepts, and learn ER modeling.

2.1 Data Models Overview

Types of Data Models:
  • Hierarchical: Tree structure (parent-child)
  • Network: Graph structure
  • Relational: Tables with relationships
  • Entity-Relationship: Conceptual modeling
  • Object-oriented: Objects with attributes/methods

2.2 Relational Model

Core Concepts:
  • Relation: Table with rows and columns
  • Tuple: Row/record
  • Attribute: Column/field
  • Domain: Set of allowable values
  • Key Constraints: Primary, foreign, candidate keys
