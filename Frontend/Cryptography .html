Cryptography and Network Security

Course Code: PCC
Credits: 4
L-T-P: 3-1-0

Course Learning Objectives

Understanding Cryptography Fundamentals

Gain a solid foundation in cryptographic principles, algorithms, and their mathematical underpinnings.

Secure Communication

Learn techniques to secure messages over insecure channels using various cryptographic methods.

CIA Triad

Understand how to maintain Confidentiality, Integrity, and Availability (CIA) of data in networked systems.

Network Security Protocols

Study various protocols designed to protect against network threats and vulnerabilities.

Why Cryptography Matters

Cryptography is essential for:

  • Protecting sensitive information (banking, healthcare, government)
  • Ensuring privacy in digital communications
  • Verifying authenticity of digital documents
  • Securing online transactions
  • Protecting against identity theft

Unit I: Introduction to Cryptography (12 Contact Hours)

Security Attacks, Services and Mechanisms

Understanding different types of security attacks (passive, active), security services (authentication, access control), and security mechanisms (encryption, digital signatures).

Conventional Encryption

Study of symmetric encryption model where the same key is used for encryption and decryption.

Symmetric Encryption Demo

Symmetric encryption uses the same key for both encryption and decryption:

Plaintext → [Encryption] → Ciphertext → [Decryption] → Plaintext
               ↑                    ↑
               Key                  Key

Common symmetric algorithms: AES, DES, 3DES, Blowfish

Classical Encryption Techniques

Substitution ciphers (Caesar, monoalphabetic, polyalphabetic) and transposition ciphers (rail fence, columnar).

Modern Block Ciphers

Principles of block ciphers, Shannon's theory of confusion and diffusion, Feistel structure.

DES and AES

Detailed study of Data Encryption Standard (DES) and Advanced Encryption Standard (AES).

Confidentiality Using Conventional Encryption

Techniques to maintain confidentiality using symmetric encryption, including traffic confidentiality and key distribution challenges.

Random Number Generation

Importance of random numbers in cryptography and methods for generating cryptographically secure random numbers.

Mathematical Foundations

Introduction to groups, rings, and fields; prime and relative prime numbers; modular arithmetic.

Unit III: Public Key Cryptography (11 Contact Hours)

Principles of Public Key Cryptosystems

Concept of asymmetric cryptography with public and private key pairs.

RSA Algorithm

Detailed study of the RSA algorithm including key generation, encryption, and decryption processes.

RSA Key Generation Steps

  1. Choose two large prime numbers p and q
  2. Compute n = p × q
  3. Compute φ(n) = (p-1)(q-1)
  4. Choose e such that 1 < e < φ(n) and gcd(e, φ(n)) = 1
  5. Compute d such that d × e ≡ 1 mod φ(n)
  6. Public key: (e, n), Private key: (d, n)

Diffie-Hellman Key Exchange

Method for securely exchanging cryptographic keys over a public channel.

Elliptic Curve Cryptography

Introduction to ECC and its advantages over traditional public key systems.

Message Authentication and Hash Functions

Authentication requirements, functions, MAC, hash functions, and security considerations.

Unit IV: Hash Algorithms and Digital Signatures (11 Contact Hours)

MD5 and SHA

Study of MD5 message digest algorithm and Secure Hash Algorithm (SHA) family.

Digital Signatures

Concept of digital signatures, authentication protocols, and Digital Signature Standard (DSS).

Authentication Applications

Kerberos authentication system and X.509 directory authentication service.

Email Security

Pretty Good Privacy (PGP) and S/MIME standards for secure email communication.

PGP Encryption Process

  1. Sender creates a random session key
  2. Message is encrypted with the session key using symmetric encryption
  3. Session key is encrypted with recipient's public key
  4. Encrypted message and encrypted session key are sent together
  5. Recipient decrypts session key with their private key
  6. Message is decrypted with the session key

Unit V: IP Security (9 Contact Hours)

IPSec Architecture

Authentication Header and ESP

Security Associations

Unit VI: Web and System Security (9 Contact Hours)

Web Security

Secure Socket Layer (SSL) and Transport Layer Security (TLS) protocols.

Secure Electronic Transaction

SET protocol for secure credit card transactions over the Internet.

System Security

Intruders, viruses, worms, trojans, and intrusion detection systems (IDS).

Types of Malware

  • Virus: Self-replicating code that attaches to clean files
  • Worm: Standalone malware that replicates to spread
  • Trojan: Disguised as legitimate software
  • Ransomware: Encrypts files and demands payment
  • Spyware: Secretly monitors user activity
  • Adware: Displays unwanted advertisements
