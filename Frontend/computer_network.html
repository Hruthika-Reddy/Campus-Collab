Unit - I: Introduction and Physical Layer

Introduction: Network Hardware, Network Software, References Models

Computer networks are interconnected devices that can communicate and share resources. Key components include:

  • Network Hardware: Devices such as routers, switches, hubs, bridges, and network interface cards (NICs) that facilitate communication.
  • Network Software: Protocols, applications, and operating system components that govern network operations.
  • Reference Models: Conceptual frameworks like the OSI (Open Systems Interconnection) model and the TCP/IP model that provide a layered approach to network design and understanding.

Physical Layer-Guided medium and unguided medium, topologies

The Physical Layer deals with the transmission of raw bits over a physical medium:

  • Guided Media: Physical cables that guide the signal, such as:
    • Twisted Pair Cable (UTP, STP)
    • Coaxial Cable
    • Fiber Optic Cable
  • Unguided Media: Wireless transmission through air or space, such as:
    • Radio Waves
    • Microwaves
    • Infrared
  • Network Topologies: The physical or logical arrangement of network nodes:
    • Bus Topology
    • Star Topology
    • Ring Topology
    • Mesh Topology
    • Tree Topology
    • Hybrid Topology

Unit - II: The Data Link Layer

The Data Link Layer : Data link Layer Design Issues, Error Control

The Data Link Layer is responsible for reliable node-to-node data transfer between two directly connected nodes:

  • Data Link Layer Design Issues:
    • Framing: Dividing the stream of bits from the Physical Layer into manageable frames.
    • Error Control: Detecting and correcting errors that may occur during transmission.
    • Flow Control: Preventing a fast sender from overwhelming a slow receiver.
  • Error Control: Techniques to handle errors:
    • Error Detection: Using codes like parity bits, checksums, and Cyclic Redundancy Checks (CRCs) to detect errors.
    • Error Correction: Using forward error correction (FEC) codes to correct errors without retransmission.

Elementary Data Link Protocols, Sliding Window Protocols

  • Elementary Data Link Protocols: Simple protocols for data transfer:
    • Unrestricted Simplex Protocol
    • Simplex Stop-and-Wait Protocol
    • Simplex Protocol for a Noisy Channel
  • Sliding Window Protocols: More efficient protocols that allow multiple frames to be in transit:
    • Go-Back-N
    • Selective Repeat

The Medium Access Control Sublayer: The Channel allocation Problem, Multiple Access protocols, Ethernet - Ethernet Cabling, standard Ethernet , Switched Ethernet, Fast Ethernet

The Medium Access Control (MAC) sublayer governs how multiple nodes share a common transmission medium:

  • The Channel Allocation Problem: How to fairly and efficiently allocate a shared communication channel among multiple users.
  • Multiple Access Protocols: Protocols used by nodes to gain access to the shared medium:
    • Random Access Protocols (e.g., ALOHA, Slotted ALOHA, CSMA, CSMA/CD, CSMA/CA)
    • Controlled Access Protocols (e.g., Reservation, Polling, Token Passing)
    • Channelization Protocols (e.g., FDMA, TDMA, CDMA)
  • Ethernet: A widely used LAN technology:
    • Ethernet Cabling: Different types of Ethernet cables (e.g., Cat 5e, Cat 6).
    • Standard Ethernet (10 Mbps): The original Ethernet standard.
    • Switched Ethernet: Using switches to create point-to-point connections, improving performance and reducing collisions.
    • Fast Ethernet (100 Mbps): An evolution of Ethernet offering higher speeds.

Unit - III: The Network Layer

The Network Layer: Network Layer Design Issues, Routing Algorithms, Congestion Control Algorithms

The Network Layer is responsible for routing packets from the source host to the destination host across multiple networks:

  • Network Layer Design Issues:
    • Routing: Determining the path that packets should take.
    • Addressing: Assigning unique addresses to network devices (e.g., IP addresses).
    • Forwarding: Moving packets from an input interface to the correct output interface on a router.
    • Fragmentation: Dividing large packets into smaller fragments to traverse networks with smaller MTUs (Maximum Transmission Units).
    • Quality of Service (QoS): Mechanisms to prioritize certain types of traffic.
  • Routing Algorithms: Algorithms used by routers to determine the best path for packets:
    • Distance Vector Routing (e.g., RIP)
    • Link State Routing (e.g., OSPF)
    • Path Vector Routing (e.g., BGP)
  • Congestion Control Algorithms: Mechanisms to prevent and manage congestion in the network:
    • Open-Loop Congestion Control
    • Closed-Loop Congestion Control
    • Examples: Leaky Bucket, Token Bucket, TCP Congestion Control (Slow Start, Congestion Avoidance, Fast Retransmit, Fast Recovery).

Internetworking, subnetting, The Network Layer in the Internet

  • Internetworking: Connecting different types of networks to form a larger network.
  • Subnetting: Dividing a larger network address space into smaller subnetworks to improve organization and efficiency.
  • The Network Layer in the Internet:
    • IP Protocol (IPv4 and IPv6): The primary protocol of the Internet's Network Layer, responsible for addressing and routing.
    • ICMP (Internet Control Message Protocol): Used for error reporting and network diagnostics.
    • ARP (Address Resolution Protocol): Maps IP addresses to MAC addresses within a local network.
    • RARP (Reverse Address Resolution Protocol): Maps MAC addresses to IP addresses (obsolete).

Unit - IV: The Transport Layer

The Transport Layer: The Transport Service, Elements of Transport Protocols, The Internet Transport Protocols: UDP, The Internet Transport Protocols: TCP

The Transport Layer provides end-to-end communication between applications running on different hosts:

  • The Transport Service: Provides services to the Application Layer, such as:
    • Process-to-Process Delivery: Delivering data to the correct application port.
    • Segmentation and Reassembly: Dividing application data into segments and reassembling them at the destination.
    • Connection Management: Establishing, maintaining, and terminating connections (in connection-oriented protocols).
    • Reliable Data Transfer: Ensuring that data is delivered correctly and in order (in reliable protocols).
    • Flow Control: Preventing a sender application from overwhelming a receiver application.
  • Elements of Transport Protocols: Common mechanisms used in transport protocols:
    • Addressing (Ports)
    • Connection Establishment and Release
    • Flow Control
    • Buffering
    • Multiplexing and Demultiplexing
    • Crash Recovery
  • The Internet Transport Protocols: UDP (User Datagram Protocol): A connectionless, unreliable transport protocol. It provides minimal overhead and is suitable for applications where speed is more important than reliability (e.g., streaming, online gaming).
  • The Internet Transport Protocols: TCP (Transmission Control Protocol): A connection-oriented, reliable transport protocol. It provides features like ordered delivery, error detection and retransmission, and flow control, making it suitable for applications that require reliable data transfer (e.g., web browsing, email, file transfer).

Unit - V: The Application Layer

The Application Layer: DNS - The Domain Name System, Electronic Mail. The World Wide Web, file transfer protocol Multimedia

The Application Layer is the top layer of the TCP/IP model and provides services directly to user applications:

  • DNS (The Domain Name System): A hierarchical and distributed naming system for computers, services, or any resource connected to the Internet or a private network. It translates human-readable domain names into IP addresses.
  • Electronic Mail (Email): A method of exchanging digital messages over the Internet or other computer networks. Key protocols include SMTP (Simple Mail Transfer Protocol) for sending, POP3 (Post Office Protocol version 3) and IMAP (Internet Message Access Protocol) for receiving.
  • The World Wide Web (WWW): An information system where documents and other web resources are identified by URLs (Uniform Resource Locators), interlinked by hypertext links, and accessible via the Internet. Key protocols include HTTP (Hypertext Transfer Protocol) and HTTPS (HTTP Secure).
  • File Transfer Protocol (FTP): A standard network protocol used for the transfer of computer files between a client and server on a computer network.
  • Multimedia: Protocols and technologies for transmitting and handling various forms of media, such as audio and video, over the network. Examples include RTP (Real-time Transport Protocol) and RTSP (Real-time Streaming Protocol).

Unit – VI: Network Security

Network Security: Cryptography, Symmetric - Key Algorithms, Public - Key Algorithms, Digital Signature

Network Security encompasses measures taken to protect data and systems from unauthorized access, use, disclosure, disruption, modification, or destruction over a computer network.

  • Cryptography: The practice and study of techniques for secure communication in the presence of adversaries. It involves encryption (converting data into an unreadable format) and decryption (converting it back).
  • Symmetric-Key Algorithms: Encryption algorithms that use the same key for both encryption and decryption. Examples include:
    • DES (Data Encryption Standard)
    • 3DES (Triple DES)
    • AES (Advanced Encryption Standard)
  • Public-Key Algorithms (Asymmetric-Key Algorithms): Encryption algorithms that use a pair of keys: a public key for encryption and a private key for decryption. The public key can be freely distributed, while the private key must be kept secret. Examples include:
    • RSA (Rivest–Shamir–Adleman)
    • ECC (Elliptic Curve Cryptography)
  • Digital Signature: A mathematical scheme for verifying the authenticity and integrity of a digital message or document. It uses public-key cryptography, where the sender signs the message with their private key, and the receiver verifies the signature using the sender's public key.
