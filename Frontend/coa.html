Computer Organization and Architecture

UNIT I: Basic Computer Structure (8 Contact Hours)

Functional Blocks of a Computer

Central Processing Unit (CPU): The "brain" of the computer that performs arithmetic, logic, and control operations.
Components: ALU, Control Unit, Registers

Memory Subsystem

  • Primary Memory (RAM/ROM)
  • Secondary Storage (HDD/SSD)
  • Cache Memory
  • Registers

I/O Subsystem

  • Input Devices (Keyboard, Mouse)
  • Output Devices (Monitor, Printer)
  • I/O Controllers
  • Buses

Data Representation

Number Systems

System Base Digits
Binary 2 0,1
Octal 8 0-7
Decimal 10 0-9
Hexadecimal 16 0-9,A-F

Signed Number Representations

  • Sign-Magnitude: MSB represents sign (0=+, 1=-)
  • 1's Complement: Invert all bits for negative numbers
  • 2's Complement: Invert and add 1 (most common)

Floating Point Representation (IEEE 754)

32-bit Single Precision Format:

Component Bits
Sign 1
Exponent 8 (biased by 127)
Mantissa 23

Example: -13.625 = 1 10000010 10110100000000000000000

UNIT II: ALU and Instruction Set Architecture (9 Contact Hours)

Arithmetic Logic Unit (ALU)

ALU Block Diagram

Performs arithmetic and logical operations on binary data

Integer Arithmetic

  • Addition: Full adder circuits
  • Subtraction: 2's complement addition
  • Multiplication: Shift-and-add algorithm
  • Division: Restoring/non-restoring algorithm

Floating Point Arithmetic

  • More complex due to exponent and mantissa handling
  • Special cases: NaN, Infinity, Denormal numbers
  • IEEE 754 standard defines operations

Instruction Set Architecture (ISA)

ISA: Interface between software and hardware defining the instructions a processor can execute.

Instruction Execution Cycle

  1. Fetch: Get instruction from memory
  2. Decode: Determine operation
  3. Execute: Perform operation
  4. Store: Write results

Addressing Modes

Mode Example Description
Immediate MOV R1, #5 Operand is in instruction
Register ADD R1, R2 Operand in register
Direct LOAD R1, 100 Address in instruction
Indirect LOAD R1, (R2) Register contains address

RISC Architecture

  • Reduced Instruction Set Computer
  • Simple, fixed-length instructions
  • Load/store architecture
  • Many registers
  • Example: ARM, MIPS

CISC Architecture

  • Complex Instruction Set Computer
  • Variable-length instructions
  • Memory-memory operations
  • Fewer registers
  • Example: x86

Processor Organization

CPU Internal Structure

Execution of Complete Instruction

  1. Instruction fetch from memory
  2. Instruction decode
  3. Operand fetch
  4. Operation execution
  5. Result storage

Control Unit Design

Hardwired Control

  • Implemented with combinational logic
  • Fast but inflexible
  • Difficult to modify
  • Used in RISC processors

Microprogrammed Control

  • Implemented with microcode in control memory
  • Slower but flexible
  • Easier to modify
  • Used in CISC processors

UNIT VI: Performance Enhancement Techniques (5 Contact Hours)

Pipelining

Pipelining: Technique where multiple instructions are overlapped in execution.

5-Stage Pipeline

Stages: IF (Instruction Fetch), ID (Instruction Decode), EX (Execute), MEM (Memory Access), WB (Write Back)

Pipeline Hazards

Type Description Solution
Structural Resource conflict More hardware resources
Data Data dependency Forwarding, stalling
Control Branch instructions Branch prediction

Parallel Processing

Parallel Processing: Simultaneous use of multiple processors to solve a problem.

Cache Coherence

Problem: Multiple caches may have different copies of same memory location

Solutions:

  • Snoopy protocols (bus watching)
  • Directory-based protocols
  • MESI protocol (Modified, Exclusive, Shared, Invalid)
