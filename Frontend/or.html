Operations Research - Complete Notes

UNIT I: Introduction to Operations Research (6 Contact Hours)

Decision Making in OR

Operations Research (OR): A scientific approach to decision making that seeks to best design and operate systems under conditions requiring allocation of scarce resources.

Key characteristics of OR approach:

  • System orientation
  • Use of interdisciplinary teams
  • Application of scientific method
  • Uncovering of new problems
  • Quantification

Development and Scope of OR

Historical Development:

  • Originated during World War II for military applications
  • Post-war expansion to industrial and business problems
  • Growth with computer technology advancement

Scope of OR:

Area Applications
Finance Portfolio optimization, Risk analysis
Marketing Media selection, Product mix
Production Inventory control, Facility planning
Personnel Manpower planning, Staff scheduling

OR Methodology

  1. Problem Formulation: Define the problem clearly
  2. Model Construction: Develop mathematical model
  3. Solution: Solve using appropriate techniques
  4. Validation: Test model against real data
  5. Implementation: Apply solution in real world

UNIT II: Linear Programming Problems (9 Contact Hours)

Introduction to Linear Programming

Linear Programming (LP): Mathematical technique for maximizing or minimizing a linear function subject to linear constraints.

Standard form of LP problem:

Maximize Z = c₁x₁ + c₂x₂ + ... + cₙxₙ
Subject to:
a₁₁x₁ + a₁₂x₂ + ... + a₁ₙxₙ ≤ b₁
a₂₁x₁ + a₂₂x₂ + ... + a₂ₙxₙ ≤ b₂
...
x₁, x₂, ..., xₙ ≥ 0

Assumptions of LP:

  • Proportionality
  • Additivity
  • Divisibility
  • Certainty

Graphical Method

Steps for Graphical Solution:
  1. Formulate the LP problem
  2. Plot constraints on graph
  3. Identify feasible region
  4. Find corner points of feasible region
  5. Evaluate objective function at each corner point
  6. Select the optimal solution
Example: Maximize Z = 3x + 5y
Subject to:
x + 2y ≤ 20
3x + y ≤ 30
x, y ≥ 0

Solution: Optimal at (8,6) with Z=54

Simplex Method

Simplex Algorithm Steps:
  1. Convert to standard form
  2. Set up initial simplex tableau
  3. Identify pivot column (most negative coefficient in objective row)
  4. Identify pivot row (minimum ratio test)
  5. Perform pivot operation
  6. Repeat until no negative coefficients in objective row

Special Cases:

  • Unboundedness: Solution space is unbounded (no finite optimal solution)
  • Multiple Optima: Alternative optimal solutions exist
  • Degeneracy: Basic variable takes zero value
  • Cycling: Algorithm cycles without reaching optimal solution

UNIT III: Advanced LP Techniques (9 Contact Hours)

Big-M Method

Artificial Variables: Used when no obvious basic feasible solution exists, especially in ≥ or = constraints.
Big-M Method Steps:
  1. Add artificial variables to ≥ or = constraints
  2. Assign large penalty M in objective function for artificial variables
  3. Solve modified problem using simplex
  4. If artificial variables remain in basis with positive value → problem is infeasible

Sensitivity Analysis

Examines how changes in parameters affect optimal solution:

  • Objective Function Coefficients: Range of optimality
  • Right-hand Side Values: Shadow prices and range of feasibility
  • Adding New Constraints: Check if current solution satisfies

Duality Theory

Dual Problem: Every LP problem (primal) has corresponding dual problem with interesting economic interpretations.
Primal (Maximization) Dual (Minimization)
Objective coefficients RHS constraints
RHS constraints Objective coefficients
≤ constraints Non-negative variables
Unrestricted variables = constraints

Economic Interpretation: Dual variables represent shadow prices - marginal value of resources.

UNIT IV: Transportation and Assignment Problems (9 Contact Hours)

Transportation Problem

Transportation Model: Special LP problem to minimize transportation cost while meeting supply and demand requirements.

Standard form:

Minimize Z = ΣΣ cᵢⱼxᵢⱼ
Subject to:
Σxᵢⱼ = aᵢ (supply constraints)
Σxᵢⱼ = bⱼ (demand constraints)
xᵢⱼ ≥ 0
Solution Methods:
  1. North-West Corner Rule (initial solution)
  2. Vogel's Approximation Method (better initial solution)
  3. Modified Distribution Method (optimality test)
  4. Stepping Stone Method (optimality test)

Assignment Problem

Assignment Model: Special case of transportation problem where each source is assigned to exactly one destination (one-to-one matching).
Hungarian Method Steps:
  1. Row reduction: Subtract smallest entry in each row
  2. Column reduction: Subtract smallest entry in each column
  3. Draw minimum lines to cover all zeros
  4. If lines = n, optimal solution exists
  5. Else, modify matrix and repeat

UNIT V: Queuing Theory (7 Contact Hours)

Queuing System Elements

  • Arrival Process: Poisson distribution (λ = arrival rate)
  • Service Process: Exponential distribution (μ = service rate)
  • Queue Discipline: FIFO, LIFO, Priority, etc.
  • System Capacity: Finite or infinite

Kendall's Notation: A/B/c/K/m/Z

  • A: Arrival process
  • B: Service time distribution
  • c: Number of servers
  • K: System capacity
  • m: Population size
  • Z: Service discipline

M/M/1 Queue Model

Single server with Poisson arrivals and exponential service times:

ρ = λ/μ (utilization factor)
P₀ = 1 - ρ (probability of zero customers)
Pₙ = (1 - ρ)ρⁿ (probability of n customers)
L = λ/(μ - λ) (average customers in system)
W = 1/(μ - λ) (average time in system)
Lq = λ²/[μ(μ - λ)] (average queue length)
Wq = λ/[μ(μ - λ)] (average waiting time)

Finite Population Models: Limited customer source affects arrival rate as more customers are in system.

Queuing Cost Models

Total Cost = Service Cost + Waiting Cost

TC = cₛ × s + cₙ × L

Where:
cₛ = cost per server per time
s = number of servers
cₙ = cost per customer per time in system
L = average number in system

UNIT VI: Inventory Models (7 Contact Hours)

Inventory Costs

  • Holding Costs: Storage, insurance, opportunity cost
  • Ordering Costs: Paperwork, delivery, handling
  • Shortage Costs: Lost sales, customer goodwill
  • Purchase Costs: Per unit cost of items

Deterministic Models

EOQ (Economic Order Quantity) Model:

EOQ = √(2DS/H)
where:
D = Annual demand
S = Ordering cost per order
H = Holding cost per unit per year

EPQ (Economic Production Quantity): Similar to EOQ but considers production rate.

Stochastic Models

Models with probabilistic demand and/or lead time:

  • Single Period Model: Newsvendor problem
  • Continuous Review (Q,r) Model: Order Q when inventory reaches r
  • Periodic Review (s,S) Model: Order up to S at fixed intervals if below s

Service Level Measures:

  • Probability of no stockout
  • Fill rate (fraction of demand met from stock)
  • Cycle service level
